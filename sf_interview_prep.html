Security & Sharing Questions
Most asked topic across ALL companies — know every layer
🔥 Most Frequent Interview TopicNearly every interview includes security scenario questions. The classic format: "User A can see X records but User B can't — same profile, same role. Why?"
Deloitte · Techno-Managerial
User A can share case with colleague — how? Always share specific type? Complex logic?
Ad-hoc sharing: Manual Sharing (Share button on record) — gives read-only or read/write.
Always share specific type: Criteria-based Sharing Rule — "Share all Cases of Type=X with Role Y".
Complex logic: Apex Managed Sharing — programmatic sharing using the __Share object with custom Sharing Reasons.
Deloitte · Technical
Same profile + role, one user sees 10 records, other sees 100. Why?
Possible reasons (pick any that apply):
• Different record ownership (OWD = Private, owner sees own records)
• Manual Sharing was applied to one user
• Criteria/Owner-based Sharing Rule applies to one
• One user has View All / Modify All via Permission Set
• Restriction Rules applied to one user
• Account Team / Opportunity Team membership
• Higher Role Hierarchy position (but they said same role, so unlikely)
General
Ownership changed on Account (OWD=Private). Which of 5 persons still has access?
After ownership change:
• Person 1 (old owner): ❌ Loses access (OWD=Private, no longer owner)
• Person 2 (manager of old owner): ❌ Unless also manager of new owner
• Person 3 (criteria-based sharing): ✅ May keep if criteria still true
• Person 4 (manual sharing): ❌ Manual sharing is deleted on ownership change
• Person 5 (Apex sharing): ✅ May keep if Apex sharing reason still meets criteria

More Interview Scenarios

General
1000 users, all with CRUD at profile level. Want to give one user Read-Only only. How?
Important correction from mock interview sheet: Permission Sets only ADD permissions — they can't negate profile-level access. To restrict one user:
• Create a new Profile with Read-only access on that object
• Assign only that user to the new profile
• Or use Restriction Rules (available from Winter '22) to limit record visibility further
General
Two users, same profile, both can READ Contact. A can't see B's records and B can't see A's. Cause?
OWD is Private. They own different records. Since OWD = Private and there's no sharing rule/manual sharing between them, each only sees their own records. They are on the same role level, so role hierarchy doesn't grant access either.

Deloitte

Key Interview Scenarios

Deloitte (3 Exp)
Need to sync 1 million Contact records to external system nightly. What approach?
Use Batch Apex with scope of 2000. Implement Database.AllowsCallouts in execute() for HTTP callouts per batch. Schedule using Scheduled Apex with a cron expression for off-peak hours (e.g., 0 0 2 * * ? for 2 AM daily). Include error handling with partial processing (Database.insert(list, false)) and log failures.
Deloitte (2nd Round)
99 SOQL in 1st chunk of Batch — what happens in next chunk?
Each Batch execute() call is a separate transaction. The governor limits reset completely for each chunk. So the 2nd chunk starts fresh with 0/100 SOQL used. However, if the 1st chunk consistently uses 99, you're dangerously close to the limit and should optimize your queries (move SOQL outside loops, use Maps).
General
Batch class is running on Account (inserting records), trigger fires, trigger has future method. What happens?
The trigger fires during batch execute(). The future method is queued. However — you cannot call batch from a future method, but calling a future FROM a batch (via trigger) is allowed. The future method runs asynchronously after the batch chunk completes. Watch for: if the trigger fires 200 times (batch scope), you'd enqueue 200 future calls — which could hit the 50 future calls per transaction limit!

Integration Questions
REST, SOAP, OAuth, Named Credentials, callout patterns
QuestionKey Points
How to expose Apex as REST API?@RestResource(urlMapping='/endpoint') + @HttpGet/@HttpPost etc.
How to consume external web services?HttpRequest, HttpResponse, Http classes. Set endpoint, method, headers, body. Send.
REST vs SOAP?REST: JSON, lightweight, URL-based. SOAP: XML, WSDL contract, POST only, legacy systems.
Where to store integration credentials?Named Credentials (URL + auth). Custom Metadata (endpoint configs). Never hardcode in Apex.
OAuth 2.0 — minimum callouts to send data?2 callouts minimum: 1 for access token + 1 for actual API call. 3 if refresh token needed.
Callout after DML — possible?No in same transaction. Use @future or Queueable to separate contexts.
What is Fire and Forget?Send data to external system, don't wait for response. Use Outbound Message or Platform Events.
Token expires — how to handle?Catch 401 response, re-authenticate (callout to token endpoint), retry original request. Store token in Custom Settings/Cache.
Can we use Navigation API for external URL?Yes — standard__webPage via NavigationMixin. But for callouts, use HttpRequest.
Inbound vs Outbound integration?Inbound: External system calls SF API (REST/SOAP). Outbound: SF calls external via callout/Platform Events.
How to handle API rate limits?Exponential backoff, retry logic, monitor response headers (X-RateLimit headers), cache responses.
What is Polling in integration?Periodically querying external system for updates. Use Scheduled Apex + Queueable for retry logic.
Veeva CRM Style
Given JSON with nested arrays — write Apex inner class structure and deserialize
For complex nested JSON: create wrapper classes mirroring the JSON structure. Use JSON.deserialize(jsonString, OuterClass.class) for typed deserialization. For unknown structure: use JSON.deserializeUntyped(jsonString) returning Map<String, Object> and cast manually. Key rule: match JSON keys exactly to Apex property names (or use @JsonAccess annotation for renaming).

Build Service Center search UI with Google Maps, real-time filtering, no downtime for new centers

Deloitte · Scenario
Email arrives in Salesforce → create Case (In-Progress, Medium) → assign to queue → check if Contact with same email exists → link or create
Option 1 (Flow): Email-to-Case → Record-Triggered Flow on Case (after insert). Get email from Case.Email__c. SOQL Contact by email. Decision: found → update ContactId on case. Not found → Create Records (new Contact) + update Case.ContactId.
Option 2 (Apex Trigger): After-insert trigger on Case. Map<String,Contact> keyed by email. One SOQL outside loop for all emails. DML only if new contact needed. Link case to contact.
LTI · Architecture
Amazon app: Items, Add Items, Cart objects. What relationships? Junction object? Calculate total? Delete behavior?
Relationships: Cart (master) → Cart_Items__c (detail, M-D). Item__c is independent (acts as product catalog). Cart_Item__c has lookup to Item__c (to get price/name).
Junction Object: Cart_Item__c serves as the junction between Cart and Items (many carts can have same item).
Total Calculation: Roll-Up Summary on Cart: SUM(Cart_Item__c.Price__c) where Status='Active'.
Delete behavior: Deleting a Cart_Item record (M-D child) does NOT delete Item__c (it's a lookup). Items remain in catalog. The cart total auto-updates via Roll-Up.
General · Data Migration
Import Account, Opportunity, Contact from Excel (old org) to new org. What tool? How to avoid errors?
Tool: Data Loader (for large volumes, full control) or Salesforce Import Wizard (simpler, limited).
Order: 1. Import Accounts first → get new IDs. 2. Map old Account IDs to new IDs. 3. Import Contacts with AccountId mapped to new IDs. 4. Import Opportunities with AccountId + Contact Role.
External IDs: Create an External_ID__c field on each object. Store old org IDs. Use Upsert with External ID to prevent duplicates.
Error avoidance: Run in sandbox first. Use partial processing. Check required fields, picklist values, field lengths, and relationship mappings before bulk import.
Deloitte · Coding
Write Batch on User to update related Account and Opportunity
global class UserAccountBatch implements Database.Batchable<SObject> {
  global Database.QueryLocator start(Database.BatchableContext bc) {
    return Database.getQueryLocator([SELECT Id, AccountId FROM User WHERE IsActive=true]);
  }
  global void execute(Database.BatchableContext bc, List<User> users) {
    Set<Id> accIds = new Set<Id>(); for(User u:users) accIds.add(u.AccountId);
    List<Account> accs = [SELECT Id FROM Account WHERE Id IN :accIds];
    // update accounts and related opportunities
    update accs;
  }
  global void finish(Database.BatchableContext bc) { /* send email */ }
}
